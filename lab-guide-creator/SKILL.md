# Lab Guide Creator

根据参考实验指导书的格式，结合指定课程目录的内容，自动生成新的实验任务指导书（.md 文件）。

## 工作流程

### 第一步：理解参考模板

1. 读取用户提供的参考实验指导书文件（路径如 `实验一.md` 或类似文件）
2. 分析模板结构，记录以下要素：
   - 标题格式（如 `# 实验任务X`）
   - 实验目的的描述方式
   - 实验内容如何组织（任务数量、编号方式）
   - 实验步骤的格式（是否包含 TODO 代码块、是否包含参考答案）
   - 实验分析的要求
   - 代码部分的格式（是否有待补充的模板代码、参考答案的格式）
   - 输出文件的命名规范

### 第二步：探索课程内容

1. 确定目标课程的目录路径。如果用户在消息中提供了具体路径，直接使用；如果用户只说课程名称，在当前工作目录中搜索匹配的数字编号目录（如 `04_函数输出`、`04-FunctionReturns`、`04` 等）。
2. 对每个目标 lecture，执行以下操作：
   - 读取主要的 `.sol` 合约文件，理解核心代码示例
   - 读取对应的 `.md` 文档（通常是 README 或同名的 .md 文件），理解知识点
   - 使用 Explore agent 并行探索多个 lecture 以提高效率
3. 整理每个 lecture 的：
   - 主题和核心概念
   - 主要代码示例
   - 关键语法点

### 第三步：设计任务结构

根据课程内容设计任务数量和主题，通常：
- 每个核心知识点对应一个任务
- 每个任务包含：实验目的说明 + 具体的操作步骤
- 任务之间应有递进关系
- 整体应覆盖参考模板中的各个章节（实验目的、内容、步骤、分析、代码、参考答案）

### 第四步：生成文档

按照参考模板的格式生成完整文档，包含以下章节：

```
# 实验任务X：<主题>

## 一、实验目的
列出本次实验的学习目标，使用"掌握"、"理解"、"了解"等动词。

## 二、实验内容
简述本次实验包含哪几个任务，每个任务的核心内容。

## 三、实验步骤
### 任务一：<任务名称>
#### 任务描述
说明这个任务要做什么。
#### 操作步骤
1. 步骤1
2. 步骤2
...

## 四、实验分析
要求学生总结的内容，包括：
- 核心概念理解
- 常见错误分析
- 关键知识点对比

## 五、参考代码
### 任务一：待补充代码
```solidity
// TODO: 代码模板
```
### 任务一：参考答案
```solidity
// 完成后的代码
```
```

### 第五步：输出

将生成的完整 .md 文件写入到与参考实验指导书相同的目录下。
文件名按照参考文件的编号规则（如 `实验一.md` → `实验二.md`）。

## 输出质量要求

- **格式一致**：严格遵循参考模板的结构和格式
- **内容准确**：代码示例必须能通过对应版本 Solidity 编译
- **步骤清晰**：每个操作步骤都是具体可执行的
- **中英混排**：中文标点 + 英文代码混合排版美观
- **代码完整**：所有 Solidity 代码块必须包含 `// SPDX-License-Identifier: MIT` 和 `pragma solidity ^0.8.34;`

## 快速参考

| 参考要素 | 常见模式 |
|---------|---------|
| 实验目的 | 使用"掌握/理解/了解 + 具体知识点"句式 |
| 实验内容 | 列出 N 个任务，说明每个任务练什么 |
| 实验步骤 | 每个任务包含：描述 + 操作步骤（编号列表）|
| 代码模板 | 在参考答案上方放置含 `// TODO:` 的待完成版本 |
| 参考答案 | 完整的、可直接编译通过的 Solidity 代码 |
| 实验分析 | 概念对比 + 常见错误 + 关键规则 |
